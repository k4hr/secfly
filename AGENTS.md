# Правила работы с SecFly

SecFly предназначен только для исследования безопасного виртуального моделирования. Перед изменениями изучите `docs/adr`, `docs/architecture`, `docs/safety` и текущий `git status`.

## Обязательные правила

- Весь пользовательский интерфейс и человекочитаемые сообщения пишутся на русском языке.
- Модуль контроля безопасности — единственный будущий арбитр смены режима; обход запрещён.
- Реальные аппаратные интеграции, координаты, эксплуатационные параметры и исполнительные команды запрещены.
- Предметные пакеты получают время через `Clock`; прямое системное время в критических пакетах запрещено.
- Решение должно быть записано до эффекта; ошибка записи запрещает эффект.
- Соблюдайте граф из `docs/architecture/dependency-boundaries.md`.
- Существенные решения оформляйте ADR и обновляйте документацию.
- Запускайте `pnpm check` и `pnpm security:check` перед завершением работы.
- После назначенного этапа остановитесь.
- Нельзя реализовывать функциональность следующего этапа заранее, даже если она кажется простой.

Нельзя добавлять цели, преследование, вооружение, применение силы, противодействие подавлению, реальную авионику или аппаратные адаптеры.
