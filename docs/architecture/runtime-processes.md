# Процессы ЭТАПА 2

- `ground-control`: русскоязычная страница `/simulation`, периодический опрос и SVG синтетической плоскости; вычислений модели нет.
- `onboard-core`: Fastify-оболочка и `/health`; смена режимов отсутствует.
- `simulator`: владелец упрощённой модели, жизненного цикла, `VirtualClock`, журнала и проверяемого API.
- `telemetry-gateway`: Fastify-оболочка для будущих синтетических сообщений.
- `scenario-runner`: CLI-команды демонстрации, ручного шага и чтения синтетического состояния.

Ground Control обращается к Simulator через серверный прокси того же происхождения. Шлюз на этом этапе не добавляется в путь данных: он не даёт предметной пользы и остаётся безопасной оболочкой. Потеря связи и неисправности не моделируются.
