# ADR-0001: Границы проекта AeroGuard

- Статус: proposed
- Дата: 2026-07-21
- Этап: PHASE 0 — Architecture

## Контекст

AeroGuard Autonomy Core — simulation-only платформа для исследования отказоустойчивого управления виртуальным гражданским мультикоптером при потере связи, недоступности GNSS, деградации синтетических датчиков, низком заряде и программных отказах.

Архитектура должна позволять детерминированно воспроизводить сценарии, объяснять каждое решение автомата и проверять безопасные режимы HOLD, RETURN и LAND. Она не является реальной авионикой и не должна вырабатывать сигналы для физических исполнительных устройств.

## Решение

1. Система работает только с виртуальным аппаратом, синтетической системой координат и синтетическими данными.
2. Единственный компонент, разрешающий смену режима, — детерминированный Safety Engine с политикой deny-by-default.
3. Simulator принимает только абстрактные simulation intents. Интерфейсы аппаратных шин, реальных автопилотов, радиомодемов и исполнительных механизмов отсутствуют.
4. Переходы HOLD, RETURN и LAND обозначают только поведение упрощённой 2D/2.5D модели и не содержат эксплуатационных параметров реального аппарата.
5. Выходы ML или эвристик, если они появятся в исследованиях, могут быть только недоверенными наблюдениями; они не меняют режим и не создают исполнительные команды.
6. Все решения, принятые и отклонённые переходы, команды, изменения конфигурации и инъекции отказов образуют монотонный журнал событий.
7. Восстановление связи не возвращает ручное управление автоматически: требуется `RECOVERY_PENDING`, стабильное окно связи и явный подтверждённый handover.
8. Конфигурация версионируется и фиксируется на время активного прогона. Ошибка или неизвестная версия конфигурации приводит к безопасному отказу.
9. Первая версия использует внутрисистемные типизированные события; внешний брокер и Redis добавляются только при измеренной необходимости.
10. Kubernetes и интеграции с реальным оборудованием исключены из первой версии.

## Явно вне области проекта

- вооружение, оружейная полезная нагрузка и применение силы;
- выбор, классификация, сопровождение или преследование целей, людей и транспорта;
- наведение на объекты и расчёт траекторий поражения;
- функции обхода, преодоления или противодействия системам подавления;
- реальные частоты, мощности, дальности, аэродинамические коэффициенты, PID-настройки, протоколы аппаратных шин и иные эксплуатационные параметры;
- сертифицированная динамика полёта, реальная авионика и управление физическим аппаратом.

## Последствия

- Результаты пригодны для обучения, исследований автомата и воспроизводимых экспериментов, но не для реального полёта или сертификации.
- Адаптер симулятора является жёсткой границей; добавление hardware/flight-controller adapter требует нового ADR и отдельного пересмотра scope.
- Любое изменение, расширяющее запрещённую область, отклоняется на ревью.

## Критерии приёмки ADR

- Во всех публичных материалах присутствует маркировка simulation-only.
- Ни один контракт не содержит адресов физических устройств или низкоуровневых actuator-команд.
- Архитектурные тесты впоследствии запрещают импорт аппаратных адаптеров в `onboard-core` и `safety-engine`.

> This project is intended solely for civilian safety research, simulation, education, and non-weaponized autonomous systems. It must not be used for target selection, pursuit, weapon guidance, payload delivery, or autonomous use of force.
